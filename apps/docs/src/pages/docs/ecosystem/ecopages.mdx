# ecopages CLI

The `ecopages` npm package is the official and preferred command-line tool for scaffolding and running Ecopages projects.

It is best when you want:

- **Fast project creation** from templates
- **Consistent Bun commands** (`dev`, `build`, `preview`, `start`)
- **A single entrypoint** for common local workflows

<Banner type="warning">
  <Banner.Title>Draft / Experimental</Banner.Title>
  <p>The CLI is currently evolving. Keep versions pinned in team projects to ensure stable team workflows.</p>
</Banner>

## Install

You can run the CLI directly with `bunx` (no global install required):

```bash
bunx ecopages --help
```

Or install it in a project:

```bash
bun add ecopages
```

## Initialize Projects

### Interactive Init

Use the base init command when you want to pick options interactively:

```bash
bunx ecopages init
```

### One-Shot Commands (Complete)

Use these when you want a full bootstrap in one command chain.

**Basic starter** - Creates a minimal Ecopages app with default setup. Example: <a href="https://github.com/ecopages/ecopages/tree/main/examples/starter-jsx" target="_blank">starter-jsx</a>.

```bash
bunx ecopages init ecopages-app && cd ecopages-app && bun install && bun dev
```

**Blog with KitaJS** - Creates a content-focused blog using the `blog-jsx` template. Example: <a href="https://github.com/ecopages/ecopages/tree/main/examples/blog-jsx" target="_blank">blog-jsx</a>.

```bash
bunx ecopages init blog-jsx --template blog-jsx && cd blog-jsx && bun install && bun dev
```

**Blog with React** - Creates a React-powered blog with routing preconfigured. Example: <a href="https://github.com/ecopages/ecopages/tree/main/examples/blog-react" target="_blank">blog-react</a>.

```bash
bunx ecopages init blog-react --template blog-react && cd blog-react && bun install && bun dev
```

**React app with Better Auth + Drizzle** - Creates a full app starter with auth and database workflow included. Example: <a href="https://github.com/ecopages/ecopages/tree/main/examples/with-react-better-auth" target="_blank">with-react-better-auth</a>.

```bash
bunx ecopages init with-react-better-auth --template with-react-better-auth && cd with-react-better-auth && bun install && bun db:generate && bun db:migrate && bun dev
```

## When To Use `ecopages` vs JSR Packages

- Use **`ecopages` CLI** for scaffolding and local app lifecycle commands.
- Use **`@ecopages/*` packages** from JSR for framework APIs in app code.

For the complete package map, see [Ecosystem Packages](/docs/ecosystem/packages).
